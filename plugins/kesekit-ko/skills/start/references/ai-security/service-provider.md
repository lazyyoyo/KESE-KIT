# AI 서비스 제공자를 위한 보안 요구사항 및 검증항목

> 출처: 과학기술정보통신부·한국인터넷진흥원 「인공지능(AI) 보안 안내서」

## 개요

AI 서비스 제공자는 AI 모델을 활용하여 서비스를 제공하는 사업자(법인)를 의미합니다.
서비스 제공자의 보안 프레임워크는 **예방(Prevention)**과 **탐지/대응(Detection & Response)** 두 축으로 구성됩니다.

---

## 1단계: 계획 및 설계

### 1.1 AI 보안 거버넌스 체계 구축 (개발자 공통)
- 1.1.1 AI 보안 거버넌스 조직 구성
- 1.1.2 보안 정책, 절차, 프로세스 구현
- 1.1.3 보안 전문인력 확보

### 1.2 위험관리 계획 수립 (개발자 공통)
- 1.2.1 생명주기/공급망 위험요소 분석
- 1.2.2 위협 모델링 및 위험 평가
- 1.2.3 위험요소 제거·완화 방안

## 2단계: 개발(도입)

### 2.1 AI 서비스 도입 시 보안 검토
- 2.1.1 AI 모델/서비스 도입 시 보안 요구사항 검토
- 2.1.2 외부 AI 모델/API 사용 시 보안 평가
- 2.1.3 서비스 환경에 적합한 보안 아키텍처 설계

## 3단계: 운영

### 3.1 서비스 운영 보안
- 3.1.1 AI 서비스 접근통제 및 인증
- 3.1.2 입출력 데이터 필터링/검증
- 3.1.3 API 보안 (Rate Limiting, 인증, 암호화)
- 3.1.4 사용자 개인정보 보호
- 3.1.5 서비스 가용성 보장 (DDoS 대비)

### 3.2 데이터 보호
- 3.2.1 사용자 데이터 수집 최소화
- 3.2.2 데이터 저장 시 암호화
- 3.2.3 데이터 보존/삭제 정책

## 4단계: 유지보수

### 4.1 모니터링 및 대응
- 4.1.1 AI 서비스 이상 동작 모니터링
- 4.1.2 보안 사고 탐지 및 대응 체계
- 4.1.3 모델 성능 저하/드리프트 탐지

### 4.2 업데이트 관리
- 4.2.1 AI 모델 업데이트 시 보안 검증
- 4.2.2 인프라/라이브러리 보안 패치

## 5단계: 피드백

### 5.1 사용자 피드백 관리
- 5.1.1 유해/부적절 출력에 대한 사용자 신고 체계
- 5.1.2 피드백 기반 보안 개선

## 6단계: 파기

### 6.1 서비스 종료 시 보안
- 6.1.1 사용자 데이터 안전 삭제
- 6.1.2 모델/API 비활성화
- 6.1.3 서비스 종료 공지 및 데이터 이관
