| 항목 | 내용 |
|------|------|
| 발행 | 과학기술정보통신부, 한국인터넷진흥원(KISA) |

### Security for AI vs Security by AI

| 구분 | Security for AI | Security by AI |
|------|----------------|----------------|
| 목적 | AI 시스템 자체를 보호 | AI로 보안 시스템 강화 |
| 대상 | AI 모델, 데이터, 인프라 | 침입탐지, 위협예측, 자동대응 |
| 위협 | 데이터 포이즈닝, 모델 탈취, 적대적 공격 | 사이버 공격, 네트워크 침입, 악성코드 |
| 기술 | 데이터 암호화, 모델 보호, API 접근제어 | AI 기반 위협탐지, 이상행동 탐지 |

## 2. AI 보안 위협 유형

### 2.1 Prompt Injection
- **Direct**: 악의적 프롬프트로 민감 정보 추출 (jailbreaking)
- **Indirect**: 외부 입력(웹페이지 등)을 통한 간접 조작
- **플러그인 악용**: 웹사이트가 플러그인을 이용한 사기

### 2.2 민감 정보 노출 (Sensitive Information Disclosure)
- 의도치 않은 데이터 유출 (다른 사용자 데이터 노출)
- Complete Filtering 오류 (부적절 응답 필터링 우회)
- Overfitting 오류 (학습 데이터 내 민감정보 기억·출력)

### 2.3 적대적 예제 공격 (Adversarial Example)
- **회피 공격 (Evasion)**: 입력 데이터에 노이즈 추가하여 모델 오분류 유도
  - 예: 정지 표지판에 스티커 → 자율주행차가 속도제한으로 오인
- **오염 공격 (Poisoning)**: 학습 데이터에 악의적 데이터 주입
  - 예: MS Tay 챗봇 — 16시간 만에 인종차별 발언 학습

### 2.4 탐색적 공격 (Exploratory Attack)
- **데이터 추출**: 전도 공격(Inversion Attack)으로 학습 데이터 탈취
- **모델 추출**: 다수 쿼리로 모델 구조·파라미터 복원
  - 블랙박스/그레이박스 공격
  - 70초 650회 쿼리로 아마존 ML 모델 복제 가능

## 3. AI 생명주기 (6단계)

| 단계 | 목표 | 주요 활동 |
|------|------|----------|
| 1. 계획 및 설계 | 목표·성공 지표 정의 | 거버넌스 구축, 위험분석, 대응방안 |
| 2. 데이터 수집 및 준비 | 학습 데이터 확보·전처리 | 데이터 소스 정의, 보안정책 준수, 데이터 분할 |
| 3. 모델 개발 | 모델 구축·학습·평가 | 알고리즘 선택, 하이퍼파라미터 조정, 검증 |
| 4. 모델 배포 | 프로덕션 환경 통합 | 패키징, 배포, 시스템 연동 |
| 5. 모니터링 및 유지보수 | 성능 추적·보안 유지 | 모델 드리프트 탐지, 패치, 업데이트 |
| 6. 파기 | 안전한 폐기 | 데이터 백업, 잔여 데이터 유출 방지, 문서화 |

## 5. 핵심 용어

| 용어 | 정의 |
|------|------|
| Adversarial Example | 모델을 속이기 위해 의도적으로 조작된 입력 데이터 |
| Data Poisoning | 학습 데이터에 악의적 데이터를 주입하는 공격 |
| Model Extraction | 쿼리를 통해 모델의 구조·파라미터를 복원하는 공격 |
| Prompt Injection | LLM의 프롬프트를 조작하여 비정상 동작을 유도 |
| Model Drift | 배포 후 시간 경과에 따른 모델 성능 저하 |
| RAG | 검색 증강 생성 (Retrieval-Augmented Generation) |
| Pred AI | 예측형 AI — 패턴 식별, 추론 기반 |
| Gen AI | 생성형 AI — 새로운 콘텐츠 생성 |
| LLM | 대규모 언어 모델 (Large Language Model) |
| Federated Learning | 데이터를 중앙에 모으지 않고 분산 학습하는 방식 |
| Differential Privacy | 개별 데이터 식별을 방지하는 프라이버시 보호 기법 |
| Homomorphic Encryption | 암호화된 상태에서 연산 수행이 가능한 암호화 방식 |
