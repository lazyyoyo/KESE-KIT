# KESE - KISA Enhanced Security Evaluation Kit

ปลั๊กอิน Claude Code สำหรับการวิเคราะห์และประเมินช่องโหว่ของโครงสร้างพื้นฐานสารสนเทศสำคัญ (CII) และการประเมินความปลอดภัยด้าน AI ตามแนวทางของ KISA (สำนักงานอินเทอร์เน็ตและความปลอดภัยแห่งเกาหลี)

---

## ภาพรวม

KESE (KISA Enhanced Security Evaluation Kit) เป็นปลั๊กอิน Claude Code ที่มอบความสามารถในการประเมินช่องโหว่ด้านความปลอดภัยอย่างครอบคลุม ตามแนวทางของ KISA (สำนักงานอินเทอร์เน็ตและความปลอดภัยแห่งเกาหลี) รองรับทั้งการประเมินโครงสร้างพื้นฐานสารสนเทศสำคัญ (CII) และการประเมินความปลอดภัยด้าน AI

## คุณสมบัติ

| สกิล | คำอธิบาย |
|------|----------|
| `/kesekit-en:start` | เริ่มการประเมินช่องโหว่ด้านความปลอดภัยแบบเต็มรูปแบบ (CII 560+ รายการ หรือ AI Security) |
| `/kesekit-en:check` | รายการตรวจสอบความสอดคล้องด้านความปลอดภัยก่อนการปรับใช้งาน |
| `/kesekit-en:fix` | สร้างสคริปต์เสริมความแข็งแกร่งและแก้ไขด้านความปลอดภัยโดยอัตโนมัติ |
| `/kesekit-en:guide` | สร้างพรอมต์การเขียนโค้ดที่ปลอดภัยสำหรับเครื่องมือ AI |

## แนวทางที่รองรับ

### 1. โครงสร้างพื้นฐานสารสนเทศสำคัญ (CII) — 560+ รายการ

**การประเมินทางเทคนิค**
| ระบบ | รหัส | จำนวนรายการ |
|------|------|:----------:|
| เซิร์ฟเวอร์ Unix/Linux | U-01~U-67 | 67 |
| เซิร์ฟเวอร์ Windows | W-01~W-64 | 64 |
| บริการเว็บ | WEB-01~WEB-26 | 26 |
| อุปกรณ์รักษาความปลอดภัย | S-01~S-23 | 23 |
| อุปกรณ์เครือข่าย | N-01~N-38 | 38 |
| ระบบควบคุม | C-01~C-51 | 46 |
| PC | PC-01~PC-18 | 18 |
| DBMS | D-01~D-26 | 26 |
| อุปกรณ์เคลื่อนที่ | M-01~M-04 | 4 |
| Web Application | 21 รหัส | 21 |
| ระบบเสมือนจริง | HV-01~HV-25 | 25 |
| คลาวด์ | CA-01~CA-19 | 19 |

**การประเมินด้านการบริหารจัดการ**: A-1~A-127 (127 รายการ, 14 โดเมน)
**การประเมินทางกายภาพ**: P-1~P-18 (18 รายการ)

### 2. แนวทางความปลอดภัย AI — 54+ รายการ

| กลุ่มเป้าหมาย | จำนวนรายการ | วงจรชีวิต |
|---------------|:----------:|----------|
| นักพัฒนา AI | 54 | 6 ขั้นตอน (วางแผน→ข้อมูล→พัฒนาโมเดล→ปรับใช้งาน→ตรวจสอบ→ยกเลิก) |
| ผู้ให้บริการ AI | ~43 | 6 ขั้นตอน (วางแผน→พัฒนา→ดำเนินงาน→บำรุงรักษา→ตอบรับ→ยกเลิก) |
| ผู้ใช้ AI | 7 | แนวปฏิบัติด้านความปลอดภัย |

## การติดตั้ง

```bash
claude plugins install kesekit-en@kesekit
```

## วิธีใช้งาน

```bash
# เริ่มการประเมินความปลอดภัยแบบเต็มรูปแบบ
/kesekit-en:start

# เรียกใช้รายการตรวจสอบก่อนปรับใช้งาน
/kesekit-en:check

# สร้างสคริปต์เสริมความแข็งแกร่ง
/kesekit-en:fix

# รับพรอมต์การเขียนโค้ดที่ปลอดภัย
/kesekit-en:guide
```

## ประวัติการเปลี่ยนแปลง

### v2.0.0 (2026-03-30)

**ปรับโครงสร้างใหม่ — นำรูปแบบ Progressive Disclosure มาใช้**

| การเปลี่ยนแปลง | ก่อนหน้า (v1.0) | หลังจาก (v2.0) |
|----------------|-----------------|----------------|
| SKILL.md | ข้อมูลทั้งหมดอยู่แบบอินไลน์ (270~465 บรรทัด) | เฉพาะเราเตอร์ (~50~80 บรรทัด) |
| แนวทาง | รองรับเฉพาะ CII | รองรับ CII + ความปลอดภัย AI |
| การจัดเก็บข้อมูล | ฝังตรงใน SKILL.md | แยกไว้ใน `references/` (18 ไฟล์) |
| รหัสรายการ | มีเฉพาะบางรายการ | ครบทุกรายการตามแนวทาง 2026 |
| ความสามารถขยาย | เพิ่มแนวทางใหม่ต้องเพิ่มจำนวนสกิล | สกิลคงที่ 4 ตัว เพิ่มเฉพาะ references |

**เพิ่มแนวทางใหม่: แนวทางความปลอดภัย AI**
- แหล่งที่มา: กระทรวงวิทยาศาสตร์และ ICT / KISA 「แนวทางความปลอดภัยปัญญาประดิษฐ์ (AI)」
- รายการตรวจสอบสำหรับนักพัฒนา AI 54 รายการ (วงจรชีวิต 6 ขั้นตอน)
- ข้อกำหนดด้านความปลอดภัยสำหรับผู้ให้บริการ AI
- แนวปฏิบัติด้านความปลอดภัยสำหรับผู้ใช้ AI 7 ข้อ

**อัปเดตแนวทาง CII**
- สกัดรายการทั้งหมดใหม่ตามคู่มือรายละเอียด 2026
- รองรับระบบรหัสรายการ (รหัสใหม่ เช่น WEB, HV, CA)

### v1.0.0 (2026-03-29)

- เปิดตัวครั้งแรก
- สกิลประเมินช่องโหว่ CII 4 ตัว (เกาหลี/อังกฤษ)
- รายการทางเทคนิค (424) + การบริหารจัดการ (127) + ทางกายภาพ (9)
