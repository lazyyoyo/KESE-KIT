# KESE - KISA Enhanced Security Evaluation Kit

Плагин Claude Code для анализа и оценки уязвимостей критической информационной инфраструктуры (CII) и оценки безопасности ИИ.

---

## Обзор

KESE (KISA Enhanced Security Evaluation Kit) — это плагин Claude Code, предоставляющий комплексные возможности оценки уязвимостей на основе руководств KISA (Корейское агентство интернета и безопасности). Поддерживает оценку как критической информационной инфраструктуры (CII), так и безопасности ИИ.

## Функции

| Навык | Описание |
|-------|----------|
| `/kesekit-en:start` | Запуск полной оценки уязвимостей безопасности (CII 560+ элементов или безопасность ИИ) |
| `/kesekit-en:check` | Контрольный список соответствия безопасности перед развёртыванием |
| `/kesekit-en:fix` | Автоматическая генерация скриптов усиления защиты и исправлений безопасности |
| `/kesekit-en:guide` | Генерация промптов для безопасного кодирования с помощью ИИ-инструментов |

## Поддерживаемые руководства

### 1. CII (Критическая информационная инфраструктура) — 560+ элементов

**Техническая оценка**
| Система | Код | Элементы |
|---------|-----|:--------:|
| Серверы Unix/Linux | U-01~U-67 | 67 |
| Серверы Windows | W-01~W-64 | 64 |
| Веб-сервисы | WEB-01~WEB-26 | 26 |
| Средства защиты | S-01~S-23 | 23 |
| Сетевое оборудование | N-01~N-38 | 38 |
| Системы управления | C-01~C-51 | 46 |
| ПК | PC-01~PC-18 | 18 |
| СУБД | D-01~D-26 | 26 |
| Мобильные устройства | M-01~M-04 | 4 |
| Веб-приложения | 21 код | 21 |
| Виртуализация | HV-01~HV-25 | 25 |
| Облако | CA-01~CA-19 | 19 |

**Административная оценка**: A-1~A-127 (127 элементов, 14 областей)
**Физическая оценка**: P-1~P-18 (18 элементов)

### 2. Руководство по безопасности ИИ — 54+ элемента

| Целевая группа | Элементы | Жизненный цикл |
|----------------|:--------:|----------------|
| Разработчик ИИ | 54 | 6 этапов (Планирование→Данные→Модель→Развёртывание→Мониторинг→Вывод из эксплуатации) |
| Поставщик услуг | ~43 | 6 этапов (Планирование→Разработка→Эксплуатация→Обслуживание→Обратная связь→Вывод из эксплуатации) |
| Пользователь | 7 | Лучшие практики безопасности |

## Установка

```bash
claude plugins install kesekit-en@kesekit
```

## Использование

```bash
# Запуск полной оценки безопасности
/kesekit-en:start

# Запуск контрольного списка перед развёртыванием
/kesekit-en:check

# Генерация скриптов усиления защиты
/kesekit-en:fix

# Получение промптов для безопасного кодирования
/kesekit-en:guide
```

## История изменений

### v2.0.0 (2026-03-30)

**Рефакторинг структуры — применение паттерна Progressive Disclosure**

| Изменение | До (v1.0) | После (v2.0) |
|-----------|-----------|--------------|
| SKILL.md | Все знания встроены (270~465 строк) | Только маршрутизатор (~50~80 строк) |
| Руководства | Только CII | CII + безопасность ИИ |
| Хранение знаний | Жёстко закодировано в SKILL.md | Вынесено в `references/` (18 файлов) |
| Коды элементов | Только часть элементов | Полный набор на основе руководства 2026 |
| Масштабируемость | Добавление руководства увеличивает количество навыков | 4 навыка фиксированы, добавляются только references |

**Новое руководство: Руководство по безопасности ИИ**
- Источник: Министерство науки и ИКТ · KISA «Руководство по безопасности искусственного интеллекта (ИИ)»
- 54 элемента проверки для разработчиков ИИ (6-этапный жизненный цикл)
- Требования безопасности для поставщиков услуг ИИ
- 7 правил безопасности для пользователей ИИ

**Обновление руководства CII**
- Полная переэкстракция элементов на основе подробного руководства 2026
- Отражение системы кодов элементов (новые коды: WEB, HV, CA и др.)

### v1.0.0 (2026-03-29)

- Первоначальный выпуск
- 4 навыка оценки уязвимостей CII (корейский/английский)
- Технические (424) + административные (127) + физические (9) элементы
