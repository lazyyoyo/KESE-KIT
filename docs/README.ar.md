# KESE - KISA Enhanced Security Evaluation Kit

اضافة لـ Claude Code لتقييم الثغرات الامنية في البنى التحتية الحيوية للمعلومات (CII) وتقييم امن الذكاء الاصطناعي، استنادا الى ارشادات وكالة الانترنت والامن الكورية (KISA).

---

## نظرة عامة

KESE (KISA Enhanced Security Evaluation Kit) هي اضافة لـ Claude Code توفر امكانيات شاملة لتقييم الثغرات الامنية بناء على ارشادات وكالة الانترنت والامن الكورية (KISA). تدعم كلا من تقييمات البنى التحتية الحيوية للمعلومات (CII) وتقييمات امن الذكاء الاصطناعي.

## الميزات

| المهارة | الوصف |
|---------|-------|
| `/kesekit-en:start` | تشغيل تقييم شامل للثغرات الامنية (CII اكثر من 560 عنصرا او امن الذكاء الاصطناعي) |
| `/kesekit-en:check` | قائمة فحص الامتثال الامني قبل النشر |
| `/kesekit-en:fix` | انشاء تلقائي لنصوص التقوية والاصلاحات الامنية |
| `/kesekit-en:guide` | انشاء موجهات البرمجة الامنة لادوات الذكاء الاصطناعي |

## الارشادات المدعومة

### 1. البنى التحتية الحيوية للمعلومات (CII) — اكثر من 560 عنصرا

**التقييم التقني**
| النظام | الرمز | العناصر |
|--------|-------|:-------:|
| خادم Unix/Linux | U-01~U-67 | 67 |
| خادم Windows | W-01~W-64 | 64 |
| خدمة الويب | WEB-01~WEB-26 | 26 |
| معدات الامن | S-01~S-23 | 23 |
| معدات الشبكة | N-01~N-38 | 38 |
| نظام التحكم | C-01~C-51 | 46 |
| الحاسب الشخصي | PC-01~PC-18 | 18 |
| DBMS | D-01~D-26 | 26 |
| الاجهزة المحمولة | M-01~M-04 | 4 |
| تطبيق الويب | 21 رمزا | 21 |
| المحاكاة الافتراضية | HV-01~HV-25 | 25 |
| السحابة | CA-01~CA-19 | 19 |

**التقييم الاداري**: A-1~A-127 (127 عنصرا، 14 مجالا)
**التقييم المادي**: P-1~P-18 (18 عنصرا)

### 2. دليل امن الذكاء الاصطناعي — اكثر من 54 عنصرا

| الفئة المستهدفة | العناصر | دورة الحياة |
|-----------------|:-------:|-------------|
| مطور الذكاء الاصطناعي | 54 | 6 مراحل (تخطيط ← بيانات ← نموذج ← نشر ← مراقبة ← ايقاف) |
| مزود الخدمة | ~43 | 6 مراحل (تخطيط ← تطوير ← تشغيل ← صيانة ← تغذية راجعة ← ايقاف) |
| المستخدم | 7 | افضل الممارسات الامنية |

## التثبيت

```bash
claude plugins install kesekit-en@kesekit
```

## الاستخدام

```bash
# بدء التقييم الامني الشامل
/kesekit-en:start

# تشغيل قائمة فحص ما قبل النشر
/kesekit-en:check

# انشاء نصوص التقوية
/kesekit-en:fix

# الحصول على موجهات البرمجة الامنة
/kesekit-en:guide
```

## سجل التغييرات

### v2.0.0 (2026-03-30)

**اعادة هيكلة البنية — تطبيق نمط Progressive Disclosure**

| التغيير | قبل (v1.0) | بعد (v2.0) |
|---------|-----------|-----------|
| SKILL.md | جميع المعارف مضمنة (270~465 سطرا) | موجه فقط (~50~80 سطرا) |
| الارشادات | CII فقط | CII + امن الذكاء الاصطناعي |
| تخزين المعرفة | مضمنة في SKILL.md | منفصلة في `references/` (18 ملفا) |
| رموز العناصر | بعض العناصر فقط | جميع العناصر بناء على دليل 2026 |
| قابلية التوسع | اضافة ارشادات جديدة تزيد عدد المهارات | 4 مهارات ثابتة، تضاف المراجع فقط |

**اضافة ارشادات جديدة: دليل امن الذكاء الاصطناعي**
- المصدر: وزارة العلوم وتكنولوجيا المعلومات والاتصالات / KISA "دليل امن الذكاء الاصطناعي"
- 54 عنصر تحقق لمطوري الذكاء الاصطناعي (دورة حياة من 6 مراحل)
- متطلبات الامن لمزودي خدمات الذكاء الاصطناعي
- 7 ممارسات امنية لمستخدمي الذكاء الاصطناعي

**تحديث ارشادات CII**
- اعادة استخراج جميع العناصر بناء على الدليل التفصيلي لعام 2026
- دمج نظام ترميز العناصر (رموز جديدة مثل WEB، HV، CA)

### v1.0.0 (2026-03-29)

- الاصدار الاولي
- 4 مهارات لتقييم ثغرات CII (كوري/انجليزي)
- عناصر تقنية (424) + ادارية (127) + مادية (9)
