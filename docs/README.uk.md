# KESE - KISA Enhanced Security Evaluation Kit

Плагін навичок Claude Code для аналізу вразливостей критичної інформаційної інфраструктури (CII) та оцінки безпеки ШІ.

---

## Огляд

KESE (KISA Enhanced Security Evaluation Kit) — це плагін Claude Code, який надає комплексні можливості аналізу вразливостей на основі настанов KISA (Корейське агентство з інтернету та безпеки). Підтримує як оцінку критичної інформаційної інфраструктури (CII), так і оцінку безпеки ШІ.

## Функції

| Навичка | Опис |
|---------|------|
| `/kesekit-en:start` | Запуск повного аналізу вразливостей безпеки (CII 560+ пунктів або безпека ШІ) |
| `/kesekit-en:check` | Контрольний список відповідності безпеці перед розгортанням |
| `/kesekit-en:fix` | Автоматична генерація скриптів зміцнення та виправлень безпеки |
| `/kesekit-en:guide` | Генерація промптів безпечного кодування для інструментів ШІ |

## Підтримувані настанови

### 1. CII (Критична інформаційна інфраструктура) — 560+ пунктів

**Оцінка технічних вразливостей**
| Система | Код | Кількість пунктів |
|---------|-----|:------:|
| Сервер Unix/Linux | U-01~U-67 | 67 |
| Сервер Windows | W-01~W-64 | 64 |
| Вебсервіс | WEB-01~WEB-26 | 26 |
| Обладнання безпеки | S-01~S-23 | 23 |
| Мережеве обладнання | N-01~N-38 | 38 |
| Система управління | C-01~C-51 | 46 |
| ПК | PC-01~PC-18 | 18 |
| СУБД | D-01~D-26 | 26 |
| Мобільний зв'язок | M-01~M-04 | 4 |
| Web Application | 21 код | 21 |
| Віртуалізація | HV-01~HV-25 | 25 |
| Хмарні технології | CA-01~CA-19 | 19 |

**Оцінка управлінських вразливостей**: A-1~A-127 (127 пунктів, 14 доменів)
**Оцінка фізичних вразливостей**: P-1~P-18 (18 пунктів)

### 2. Посібник з безпеки ШІ — 54+ пунктів

| Цільова група | Кількість пунктів | Життєвий цикл |
|---------------|:------:|---------|
| Розробник ШІ | 54 | 6 етапів (Планування→Дані→Розробка моделі→Розгортання→Моніторинг→Виведення з експлуатації) |
| Постачальник послуг ШІ | ~43 | 6 етапів (Планування→Розробка→Експлуатація→Обслуговування→Зворотний зв'язок→Виведення з експлуатації) |
| Користувач ШІ | 7 | Правила безпеки |

## Встановлення

```bash
claude plugins install kesekit-en@kesekit
```

## Використання

```bash
# Запустити повну оцінку безпеки
/kesekit-en:start

# Виконати контрольний список перед розгортанням
/kesekit-en:check

# Згенерувати скрипти зміцнення
/kesekit-en:fix

# Отримати промпти безпечного кодування
/kesekit-en:guide
```

## Структура проєкту

```
KESE-KIT/
├── .claude-plugin/
│   └── marketplace.json              ← Метадані плагіна
├── skills/                            ← Англомовні навички (маршрутизатор)
│   ├── start/
│   │   ├── SKILL.md                  ← Маршрутизатор (~80 рядків)
│   │   └── references/               ← Знання за настановами
│   │       ├── cii/                  ← CII 14 довідників
│   │       └── ai-security/          ← Безпека ШІ 4 довідники
│   ├── check/
│   ├── fix/
│   └── guide/
├── skills-ko/                         ← Корейські навички (та сама структура)
├── authorkit/                         ← Вихідні документи та робочі матеріали
│   ├── converted/
│   │   ├── ref-001/                  ← Управлінський/фізичний посібник (full.md)
│   │   ├── ref-002/                  ← Технічний посібник (full.md)
│   │   └── ref-003/                  ← Посібник з безпеки ШІ (full.md)
│   └── ...
├── 문서/                              ← Оригінальні PDF
└── README.md
```

## Історія змін

### v2.0.0 (2026-03-30)

**Рефакторинг структури — застосовано шаблон Progressive Disclosure**

| Зміна | Раніше (v1.0) | Тепер (v2.0) |
|-------|------------|------------|
| SKILL.md | Усі знання вбудовані (270~465 рядків) | Лише маршрутизатор (~50~80 рядків) |
| Настанови | Лише CII | CII + безпека ШІ |
| Зберігання знань | Жорстко закодовано в SKILL.md | Виділено в `references/` (18 файлів) |
| Коди пунктів | Включено лише деякі пункти | Усі пункти на основі посібника 2026 |
| Масштабованість | Нові настанови потребують додаткових навичок | 4 навички фіксовані, додаються лише references |

**Додано нову настанову: Посібник з безпеки ШІ**
- Джерело: Міністерство науки та ІКТ / KISA «Посібник з безпеки штучного інтелекту (ШІ)»
- 54 пункти перевірки для розробників ШІ (6-етапний життєвий цикл)
- Вимоги безпеки для постачальників послуг ШІ
- 7 правил безпеки для користувачів ШІ

**Оновлення настанов CII**
- Усі пункти повторно витягнуто на основі детального посібника 2026 року
- Відображено нову систему кодів (WEB, HV, CA та інші нові коди)

### v1.0.0 (2026-03-29)

- Перший випуск
- 4 навички аналізу вразливостей CII (корейська/англійська)
- Технічні (424) + Управлінські (127) + Фізичні (9) пункти
